URI de redirection OAuth

Dans Google Cloud Console → votre client OAuth « Web application », ajoutez exactement cette URI dans Authorized redirect URIs :

{{ gsc.redirect_uri }}

En local : ajoutez aussi http://localhost:5000/admin/gsc/callback si vous testez en développement. Activez l'API Google Search Console API sur le même projet Google Cloud.

Erreur « access_denied » / appli en test

Si Google affiche « pilotcore.fr n'a pas terminé la procédure de validation » avec 403 : access_denied, ce n'est pas un problème d'URI de redirection — l'appli OAuth est en mode Test et seuls les comptes listés peuvent se connecter.

  1. Ouvrez OAuth consent screen (même projet que le Client ID).
  2. Vérifiez que le statut est Testing (normal pour un usage admin interne).
  3. Dans Test users → Add users, ajoutez l'adresse Gmail exacte avec laquelle vous cliquez sur « Se connecter avec Google ».
  4. Connectez-vous avec ce même compte (il doit aussi avoir accès à la propriété dans Search Console).

Pas besoin de publier l'appli pour un usage solo admin. La publication + vérification Google n'est utile que si d'autres utilisateurs externes doivent autoriser l'appli.

Erreur 500 sur la page Google

Si la page accounts.google.com/.../oauth/consent affiche une erreur 500, vérifiez dans OAuth consent screen :

  1. Authorized domains : ajoutez pilotcore.fr
  2. Application home page : https://www.pilotcore.fr
  3. Privacy policy et Terms of service : URLs valides (ex. page Confidentialité du site)
  4. Sur Scalingo : PUBLIC_BASE_URL=https://www.pilotcore.fr (même host que l'URI de redirection ci-dessus)
  5. Au moment de valider, choisissez le bon compte Google (celui listé en test user et propriétaire GSC)
